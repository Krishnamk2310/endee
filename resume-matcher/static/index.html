1. The Foundation (Endee DB & FastAPI)

We have two main pieces of software running:

  • Endee Vector Database: Running inside a Docker container. This is a specialized database built in C++ that is designed to store and instantly search through massive arrays of numbers (vectors).
  • FastAPI Backend: The Python web server. This acts as the brain that coordinates data between the user and the Endee database.

2. Step One: Uploading a Resume (POST /upload_resume)

When you use the upload box to submit a PDF file, here is the sequence of events:

  • Text Extraction: The file hits our FastAPI endpoint in app.py. We pass the raw PDF data to our custom pdf_parser.py script, which uses the pypdf library to read through the document and extract all the raw, human-readable text.
  • AI Embedding: We give that extracted text to an AI model called sentence-transformers (specifically the all-MiniLM-L6-v2 model). This model understands language and translates the entire resume text into a Vector Embedding‚Äîwhich is simply a mathematical array of 384 floating-point numbers. Think of this vector as a mathematical coordinate that represents the "meaning" or "skills" described in the resume.
  • Storage (Endee API): Our vector_store.py script takes this 384-dimensional array, attaches a unique ID and some metadata, and makes a direct HTTP request to the Endee database via POST /api/v1/index/{index_name}/vector/insert.

3. Step Two: Finding a Match (POST /match)

When you have a new open role and need to find the best candidate, you use the search box:

  • Job Description Embedding: You type in the Job Description. The exact same AI model translates this Job Description text into a new 384-dimensional query vector.
  • Semantic Search (Endee API): Our vector_store.py script sends this query vector directly to the Endee Database via POST /api/v1/index/{index_name}/search to find the mathematically closest resume vectors.
  • Endee Engine at work: Because Endee is an ultra-fast vector database, it uses a distance metric called Cosine Similarity to instantly measure the angles between your Job Description vector and every single candidate's resume vector it has stored. The smaller the angle, the more semantically similar they are.
  • Returning Results: Endee packages the best matches up into a highly compressed binary format called MessagePack and sends it back to Python.
  • Final Output: Our Python script unpacks the binary response, extracts the candidate IDs, original filenames, and their Similarity Scores (a number between 0 and 1, where 1 is a perfect match), and displays them ranked in order to you.
